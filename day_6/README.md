# day_6 (first_task) (#Relational & Logical Check)

Today is the day 6 of this wonderful and continuous journey of python.
my first task for today is a relational and logical check device which basically detects that weather the input number greater than 10,less than 50 and equal to 100.

some features of this device are as follows:
1. Pure Logical Evaluation: Evaluates complex conditional logic smoothly without using standard conditional statements like if-else, showcasing a clean functional approach.
2. Smart Operator Combination: Effectively combines Relational Operators (>, <, ==) with Logical Operators (and, or) to create a robust filtering rule.
3. Boolean Output Format: Automatically processes inputs and returns a native Python boolean value (True or False) as the final result.
4. Independent Condition Blocks: Breaks down the problem statement into distinct, highly readable variables (cond_1, cond_2, cond_3) making the code incredibly easy to audit and maintain.
5. Interactive Terminal UI: Enhances user experience with a center-aligned header (.center(80)) and clean string output formatting.

# day_6 (second_task) (#Smart ATM)

Here's my second task for today. It is a smart atm device which basically calculates first of all tha total number of 500 notes in the input amount,then it gives the remain balance, after this it gives the number of 100 notes in the remain balance and finally it gives the final balance.

Some features of this device are as follows:
1. Multi-Tier Denomination Breakdown: Automatically breaks down any large currency amount into multiple note types (500 and 100) sequentially.
2. Chained Arithmetic Logic: Smartly uses the remainder (modulus) of the first calculation as the input for the next calculation, mimicking a real-world ATM algorithm.
3. Highly Efficient Math: Accomplishes multi-level division using lightweight Python operators (// and %) without relying on conditional statements or loops.
4. Clean and Descriptive Variable Naming: Uses perfectly descriptive variables like balance_after_500s and final_balance, making the code self-explanatory.
